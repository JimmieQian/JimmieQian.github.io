搭建个人博客

发表于 2017-07-23 | 分类于 博客搭建 | 阅读次数

介绍

本文是采用 github pages + hexo + 域名绑定 来搭建个人博客

记录我搭建blog的过程

Why Github Pages?

对于码农而言,GitHub无疑是很亲切的. Github Pages托管在GitHub上,使用起来很方便.

搭建环境前提

  1. nodejs 官网
  2. git 官网

对于node与git环境的搭建,这里就不在赘述

创建 GitHub Pages

GitHub Pages官网 有详细的介绍,这里简单的进行演示

1. 在GitHub上创建仓库,仓库名需要以userName.github.io来命名.

比如你GitHub上的用户名是”Jimmie”,则你需要创建一个叫”jimmie.github.io”的仓库.
(命名对大小写不敏感)

这个来自官方的截图,只要填写仓库名,其他项可以都不填,然后点击创建仓库

2. 在第一步,我们只是创建了一个仓库,里面没有任何分支,而GitHub Pages 需要在master分支上操作,所以我们需要创建一次提交.

这里使用git terminal来演示

3. 如果按照前2步的步骤,大致已经搭建完毕了.

这个时候,你可以访问 https://username.github.io. 网址,
如果成功,你讲看到 网页返回 ‘hello world’

Hexo 搭建博客

HEXO 官方文档 里面有详细的建站过程

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

1. Hexo 安装

Hexo 的搭建需要 git和nodejs支持,搭建完环境后,使用npm来安装hexo

在Mac上如果出现 { Error: Cannot find module ‘./build/Release/DTraceProviderBindings’} ,

请参考 Error with DTrace (Mac OS X)

使用 npm install hexo --no-optional 来安装hexo

2. Hexo 建站

在你的电脑上创建一个blog目录,比如 你想创建jimmie-blog根站,则

此时,你只需要开启本地服务,就算搭建完本地博客了

本地监听的是 4000 端口,打开 http://localhost:4000 就可以看到默认的博客样式

3. hexo常用指令

  • hexo clean - 清楚缓存
  • hexo generate - 生成静态文件 //简写 hexo g
  • hexo server - 启用服务器 //简写 hexo s
  • hexo deploy - 部署到仓库 //简写 hexo d
  • hexo d -g - 先生成静态文件,再部署到仓库

Hexo部署到GitHub Pages

需要使用 git ssh的方式来部署

下面会介绍 电脑绑定github的ssh的方法

1. git 设置用户名,邮箱

使用你在GitHub上的用户名和邮箱

2. 生成秘钥

如果不需要加密密码,直接连续3回车后,会在用户目录生成 .ssh文件夹,目录下id_rsa.pub就是产生的秘钥

3. 将秘钥添加到GitHub

在 GitHub -> Setting -> SSH and GPG Keys -> New SSH Key 中添加秘钥

4. 验证绑定是否成功

过程中看到 Are you sure you want to continue connecting (yes/no)?,选择yes

如果看到 Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access.,
说明已经ssh绑定成功

5. 到此git配置完成,现在需要将 Hexo 的部署绑定到 GitHub Pages

在Hexo项目的根目录下,找到 _config.yml文件,找到部署(deploy)所在位置,添加一下字段

username 改为自己的用户名即可.

6. hexo部署到GitHub,还需要部署插件.

需要在项目根目录下安装.

7. 使用 hexo d 来测试部署

如果命令返回 INFO Deploy done: git 则部署完成.

访问 “https://username.github.io/",查看是否与本地的hexo服务器内容一致.

将GitHub Pages , Hexo与域名绑定

在GitHub Pages官方有介绍如何 绑定域名的文档

1. 购买域名

建议到 godaddy ,或者 万网 上购买域名.
godaddy 是国外的域名提供商,也提供支付宝支付,购买时可以搜一些优惠码进行购买
万网 是阿里的产品

2. 域名绑定

在域名服务器的DNS管理中,添加两条A记录

3. hexo配置

需要在GitHub项目的根目录上,添加CNAME记录,但是由于hexo每次提交都会覆盖GitHub上的内容,可以在 hexo项目的source目录下添加 名为 CNAME的文件.里面的值填上你的域名,你用hexo部署的时候,就会提交到GitHub的根目录中去.

如果你填的是sample.com 无论你是否输入www, 都会定位到http://sample.com

如果你填的是www.sample.com 则定位到http://www.sample.com

hexo clean,
hexo d -g 提交到GitHub

4. 访问域名

访问你申请的域名,如 sample.com, 就可以看到效果了.

结尾

至此, 配置结束.

到 hexo-theme 去挑选你喜欢的主题,然后开启个人博客之旅吧!!!
